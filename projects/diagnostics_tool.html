Embedded Hardware / CAN Bus

Handheld Diagnostics Tool

A zero-latency manual override and diagnostic bridge for the Razorbotz NASA Lunabotics platform.

ESP32-S3 LVGL (C) CAN Bus Fusion360

The Problem: Fatal Boot Latency

In the NASA Lunabotics competition, every second of the 5-minute setup phase is critical. Last year, we faced a catastrophic failure: a linear actuator on the bucket failed, but because our onboard computers take **1.5 minutes to boot and sync to the network**, we ran out of time to diagnose the issue before our run began.

"We needed a solution that bypasses the high-level OS and talks directly to the hardware in under 1 second."

The Solution: Custom CAN Bridge

I developed a handheld tool using an **ESP32-S3 Touch** with an integrated CAN transceiver. This device allows ground crew to drive the drivetrain, excavation arm, and bucket directly via the CAN bus, bypassing the primary Jetson computer entirely during the setup phase.

Reverse Engineering the API

The primary challenge was controlling the TalonSRX and TalonFX motor controllers. Since the manufacturer (CTRE) does not publicly document their CAN API, I used the candump utility on Linux to intercept messages sent by their official software.

  • ▹ Captured known speed values to decode motor command frames.
  • ▹ Parsed returning telemetry frames to graph real-time current draw and position.
  • ▹ Implemented the protocol in pure C for the ESP32 firmware.
