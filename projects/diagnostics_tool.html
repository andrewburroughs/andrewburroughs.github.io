Handheld Diagnostics Tool

A portable CAN-based diagnostics and control tool for rapid robot inspection, local actuator testing, and field troubleshooting without requiring the main onboard computers.

C LVGL ESP32-S3 CAN Bus

Project Overview

This handheld diagnostics tool was developed for the Arkansas Razorbotz lunar excavation robot used in NASA Lunabotics. During competition, operators are separated from the arena and must rely on ground crew to perform final inspections and last-minute troubleshooting during a short setup window. Because the onboard computers required significant boot and network initialization time, diagnosing faults during that period was difficult and often impractical.

To solve that problem, I designed a self-contained handheld tool that could directly interface with the robot over CAN bus, allowing the team to command subsystems, inspect telemetry, and identify hardware faults without waiting for the full onboard autonomy stack to come online. This reduced the time required to begin diagnostics from well over a minute to nearly instantaneous local access.

System Design

The device is built around a Waveshare ESP32-S3 Touch, which includes an integrated CAN interface and enough compute capability to drive a responsive embedded graphical interface. The system communicates directly with the robot’s Talon SRX and Talon FX motor controllers, allowing operators to test the drivetrain, excavation arm, and bucket subsystems independently of the robot’s main computers.

I developed the interface using LVGL and implemented the control and telemetry handling logic in C. The tool provides direct command input, live data monitoring, and subsystem-level diagnostics in a format that can be used quickly by operators and ground crew under time pressure.

Key Features

  • ▹
    Direct CAN Control: Sends motor control commands directly over CAN bus, allowing local actuation testing without requiring the robot’s primary compute system or wireless network.
  • ▹
    Live Telemetry Display: Reads and parses returning motor controller telemetry in real time to display position, velocity, current, and system status.
  • ▹
    Subsystem Diagnostics: Enables fast isolation of faults in the drivetrain, excavation arm, and bucket mechanisms during pre-run inspection and repair.
  • ▹
    Embedded Data Visualization: Displays parsed values and graphs motor data locally on the device for quick operator interpretation.
  • ▹
    Custom Enclosure Design: Includes a purpose-built 3D-printed housing designed in Fusion 360 around the electronics and battery system for durable field use.

Challenges & Solutions

One of the most technically challenging parts of this project was communicating with CTRE Talon motor controllers at the CAN message level. Official documentation for low-level message formats was limited, and the available public resources were incomplete and based on older firmware revisions.

To overcome this, I reverse engineered the command and telemetry packets by issuing known motor commands, capturing CAN traffic with Linux tools such as candump, and correlating the observed message patterns with expected controller behavior. This allowed me to reconstruct the packet structure needed both to command the motors and to decode the returning telemetry reliably.

Mechanical Design

This project also required mechanical design work. Before building this tool, I had very little experience with CAD, so I taught myself Fusion 360 in order to create a compact enclosure around the ESP32-S3 board and the 18650 battery pack.

The resulting housing was designed for clean assembly, secure fitment, and portability in the field. The final tolerances were accurate enough that the electronics slid into place and locked securely inside the enclosure.

Outcome

The handheld diagnostics tool successfully provided direct local control and fault inspection for the robot, dramatically reducing the time needed to begin troubleshooting from roughly 100 seconds to under a second. It also improved field readiness by allowing ground crew to verify subsystem functionality before a run without depending on the full onboard software stack. The interface proved intuitive enough for rapid adoption, and the tool became a practical asset for both diagnostics and operations support.
